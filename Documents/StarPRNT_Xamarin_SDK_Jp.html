  • ESC/POS ⇔ StarPRNT エミュレーション切り替え方法

    1. プリンタの電源を投入し、プリンタカバーを開きます。

    2. 電源ボタンとFEEDボタンを同時に長押しした後、ERRORランプが5回点滅したことを確認し、すぐに電源ボタンとFEEDボタンから指を放します。エミュレーションの切り替えが自動的に行われます。

    3. 用紙をセット後、プリンタカバーを閉めると設定されたエミュレーションが印字されます。エミュレーションが正しく切り替わっていない場合、再度1~3の手順を行ってください。その際、2の手順においては、点滅中に指を放さず、点滅が5回完了したことを確認してから指を放すように注意してください。

    4. エミュレーションの切り替え後は、プリンタの電源を一度オフにしてから再投入してください。選択したエミュレーションは、プリンタの電源を再投入することで有効になります。

1.4. プリンタのデバイスへの接続

1.4.1. Bluetooth インターフェイス

mC-Print2、mC-Print3、TSP100IIIBIをデバイスと接続する方法については、オンラインマニュアル(設定する-タブレット/PCと接続する)を参照ください。
プリンタの工場出荷時の初期設定では機種ごとに"Star Micronics"等、共通のBluetooth デバイス名が設定されている場合があります。同じ Bluetooth デバイス名のプリンタを複数台配置して運用される場合、Bluetoothデバイス名の変更を行うとプリンタの判別が付けやすく便利です。

  • Bluetooth デバイス名の変更等、プリンタのBluetooth設定値は専用のソフトウェアを使用して変更することができます。各プリンタで使用するソフトウェアはプリンタ設定用ソフトウェアを参照ください。
1.4.1.1. Bluetoothプリンタとのペアリング (iOS)

  1. プリンタの電源を投入します。

  2. プリンタのセキュリティ方式がSSPの場合は、PAIRボタンを5秒以上押してペアリング可能にします。

  3. iOSデバイスにて[設定]、[Bluetooth]をタップします。

  4. [Bluetooth]を ”オン”に設定すると、iOSデバイスとペアリングが可能なBluetoothデバイスの検索を行い、表示します。ペアリングを行うプリンタをタップします。

  5. PIN Codeを使用する場合、PIN番号(パスキー)を入力します。工場出荷時の初期値は“1234”です。(SSPの場合はこの操作は必要ありません。)

  6. 以下の表示が確認できればペアリング完了です。

  • 専用のソフトウェアを使ってiOSポート名を変更することができます。必要に応じてご使用ください。各プリンタで使用するソフトウェアはプリンタ設定用ソフトウェアを参照ください。

  • iOSポート名はペアリング実行後、以下の手順で確認できます。
    「設定」-「一般」-「情報」 Bluetoothアドレスの下に表示
1.4.1.2. Bluetooth プリンタとのペアリング (Android)

  1. プリンタの電源を投入します。

  2. プリンタのセキュリティ方式がSSPの場合は、PAIRボタンを5秒以上押してペアリング可能にします。

  3. Androidデバイスにて「設定」をタップします。

  4. 「Bluetooth」を"ON"に設定します。

  5. デバイスの検索を実行します。
    Bluetoothデバイスのスキャンが行われ、見つかったデバイスが表示されます。 設定を行うプリンタのデバイス名をタップします。

  6. PIN Codeを使用する場合、PIN番号(パスキー)を入力します。工場出荷時の初期値は“1234”です。(SSPの場合はこの操作は必要ありません。)

1.4.1.3. Bluetooth プリンタとのペアリング (Windows)

  1. プリンタの電源を投入します。

  2. プリンタのセキュリティ方式がSSPの場合は、PAIRボタンを5秒以上押してペアリング可能にします。

  3. Windowsデバイスにて「設定」から「デバイス」を開きます。

  4. 「Bluetoothとその他のデバイス」を選択します。

  5. 「Bluetooth」を"ON"に設定します。

  6. 「プリンターとスキャナー」-「プリンターまたはスキャナーを追加します」を選択します。

  7. 接続するBluetoothデバイスを選択して、「デバイスの追加」を選択します。

  8. PIN Codeを使用する場合、PIN番号(パスキー)を入力します。工場出荷時の初期値は“1234”です。(SSPの場合はこの操作は必要ありません。)

  9. 接続が完了すると” ペアリング済み” と表示されます。

1.4.2. Ethernet インターフェイス

mC-Print2、mC-Print3、TSP100IIILANをデバイスと接続する方法については、オンラインマニュアル(設定する-タブレット/PCと接続する)を参照ください。
プリンタの工場出荷時の初期設定はDHCPが有効になっています。使用するネットワークがDHCPをサポートする場合、プリンタが自動的にIPアドレスを取得できるように、必要なネットワーク構成を構築してください。
また、プリンタ本体の#9100 Multi Sessionを無効に設定して使用してください。設定は、FEEDボタンを押しながら電源を投入することで実行される自己印字で確認できます。
本ソフトウェアはTCP通信ポート9100、TCP通信ポート9101、UDP通信ポート22222を使用します。

  • ＃9100 Multi Sessionの確認・変更、固定IPアドレスの設定方法については、こちらのリンクの「対応 OS・環境一覧 ＞ ユーティリティ一覧」より「イーサネットプリンタ利用手引き」をご参照ください。

  • POSプリンタのEthernet設定は専用のソフトウェアを使用して変更することができます。各プリンタで使用するソフトウェアはプリンタ設定用ソフトウェアを参照ください。
    TSP100III Utility、Star Setting UtilityはIPアドレスの割り当てられていない(IP アドレス：0.0.0.0)プリンタでは使用できません。
1.4.2.1. Ethernetプリンタとの接続設定 (iOS)

  1. プリンタにIPアドレスを割り当て、ネットワークに接続します。 .「設定」をタップします。

  2. 「Wi-Fi」を"ON"に設定します。

  3. iOSデバイスをプリンタと同じネットワークに接続します。

1.4.2.2. Ethernetプリンタとの接続設定 (Android)

  1. プリンタにIPアドレスを割り当て、ネットワークに接続します。

  2. 「設定」をタップします。

  3. 「Wi-Fi」を"ON"に設定します。

  4. Androidデバイスをプリンタと同じネットワークに接続します。

1.4.2.3. Ethernetプリンタとの接続設定 (Windows)

  1. プリンタにIPアドレスを割り当て、ネットワークに接続します。

  2. Windowsデバイスをプリンタと同じネットワークに接続します。

1.4.4. USBインターフェイス

使用しているデバイスに、プリンタドライバ等のソフトウェアがインストールされることはありません。

1.4.4.1. USBプリンタとデバイスの接続 (iOS)

  1. プリンタとiOSデバイスをLightningケーブルで接続してください。

  • USBプリンタのデバッグ方法
    USBプリンタを使用する場合、iOSデバイスのLightningポートはプリンタとの接続に使用されるため、Lightningケーブル経由でのアプリのインストールやデバッグができません。しかし、下記の条件を満たす場合はワイヤレスデバッグを使用する事が可能です。

    • macOS 10.12.4以降

    • Visual Studio for Macの最新バージョン

    • Xcode 9.0以降

    • iOS 11.0以降のデバイス

※ 下記の表記、手順等はバージョンアップにより、変更される可能性があります。最新の情報は、Microsoft社のサイトをご参照ください。

  1. iOSデバイスとmacを同一ネットワークに接続します。

  2. iOSデバイスの"設定"アプリより、iOSデバイスのパスコードを設定します。

  3. iOSデバイスとmacをLightningケーブルで接続します。

  4. Xcodeを起動し、"Window"-"Devices and Simulators"を選択します。

  5. 左ペインで使用するiOSデバイスを選択し、"connect via network" にチェックを入れます。

  6. Lightningケーブルで接続していない状態でもVisual Studio上にネットワーク上のiOSデバイスが表示され、アプリのインストールとデバッグが可能になります。
1.4.4.2. USBプリンタとデバイスの接続 (Android)

  1. プリンタとAndroidデバイスをUSBケーブルで接続してください。

  • 接続に使用するUSBケーブルはデバイスによって異なります。デバイスの仕様をご確認ください。

  • USBハブを用いて複数デバイスの接続を行う場合、カスケード接続は1段までを推奨します。また、同時に接続できるデバイス数は4デバイスを上限とします。

try
{
    port = Port.GetPort(portName, portSettings, timeout);

    IPrinterStatus printerStatus = port.GetParsedStatus();

    if(!printerStatus.Offline)
    {
      // Printer Online
      if(printerStatus.BlackMarkError)
      {
        // Black Mark Error
      }
      if(printerStatus.CompulsionSwitch)
      {
        // Cash Drawer Opened
      }
      else
      {
        // Cash Drawer Closed
      }
    }
    else
    {
      // Printer Offline
    }
}
catch (PortException exception)
{
    // Error
}
finally
{
    if (port != null)
    {
        Port.ReleasePort(port);
        port = null;
    }
}

3.1. IPortインターフェイスを使った印刷処理フロー

IPortインターフェイスを利用し、以下の手順に沿って印刷を行います。

Sample Code
IPort port = null;
IPrinterStatus printerStatus;

try
{
    // ポートオープン
    port = Port.GetPort(portName, portSettings, timeout);

    // 印刷終了監視開始
    printerStatus = port.BeginCheckedBlock();

    // 印刷前のステータス判定
    if (printerStatus.Offline)
    {
        // プリンタオフライン(用紙切れ、カバーオープンなど)
        return;
    }

    DateTime startDateTime = DateTime.Now;

    int total = 0;

    while (true)
    {
        // 印刷データ送信
        int written = port.WritePort(commands, total, commands.Length - total);

        total += written;

        // 全印刷データ送信完了
        if (total >= commands.Length)
        {
            break;
        }

        TimeSpan timeSpan = DateTime.Now - startDateTime;

        if (timeSpan.TotalMilliseconds >= 30000)
        {
            // タイムアウトエラー
            return;
        }
    }

    // 印刷終了監視終了タイムアウト時間設定
    port.EndCheckedBlockTimeoutMillis = 30000;

    // 印刷終了監視終了
    printerStatus = port.EndCheckedBlock();

    // 印刷終了監視終了時のステータス判定
    if (printerStatus.Offline)
    {
        // プリンタオフライン(用紙切れ、カバーオープンなど)
        // 印刷異常終了
    }
    else
    {
        // プリンタオンライン
        // 印刷正常終了
    }
}
catch (PortException exception)
{
    // エラー発生
}
finally
{
    // ポートクローズ
    if (port != null)
    {
        Port.ReleasePort(port);
        port = null;
    }
}
